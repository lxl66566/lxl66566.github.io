        # bot

        我曾使用 nonebot2open in new window + cqhttp 的方式搭建 bot。这种方案使用 python 语言，只能搭建 QQ bot，并且需要自己写比较底层的代码。在此感谢 yaowan233open in new window 学长的指导。

        而现在我选择 koishiopen in new window 框架。该框架基于 node.js，具有图形化的操作界面和高度预设，可以接入多个平台，也有成熟的 npm 包管理。文档虽说不算优秀，但至少比 nonebot2 的垃圾强数倍。同时也有论坛open in new window以供交流。但是代价就是 node.js 的高运行时内存占用了。（常态 600M）

        我建的 bot 有两个实例：一个是之前一直在用的 QQ bot atri（2433269451），另一个是后来使用 koishi 的 Telegram bot atri botopen in new window。

        查看 atri(QQ) 的信息

        她的功能：

        • 背单词（英日双语）
        • 随机数
        • 带饭记账
        • 偷窥我

        请加为好友（联系绝对值_x同意好友）后发送help获取详细指令。你也可以在这里open in new window得到她的实现上述功能的python源码。关于环境：

        1. 使用conda创建出所需环境（此处环境名为atri）：conda create -n atri python=3.9

        nonebot2需要在python>=3.8环境下运行

        1. 使用pip安装依赖：
        pip install nb-cli
pip install nonebot-adapter-onebot
        1. 参考随笔中的启动脚本

        # install

        这里推荐使用 npm 安装open in new window，而不是官方推荐的下载安装包。使用 npm 安装对 bot 功能没有影响。

        npm i -g create-koishi@latest
npm init koishi

        # 使用

        • telegram：本机没有公网 ip，因此选择 polling 工作模式。代理用 http 好像不能用（这也是我第一次退坑 koishi 的理由），之后尝试用 socks 端口就可以了。

        • QQ：参考论坛open in new window。回复挺破碎的，需要多翻一翻。我能够成功使用，不过会报 warning:

          由用户 '*' 发送的 session '*' 缺失 username。插件处理可能出现问题。若要针对此问题进行反馈，请前往 https://k.ilharp.cc/2510 。错误码：2130

          不影响使用，开发者也说不用在意open in new window，那就不管了。
